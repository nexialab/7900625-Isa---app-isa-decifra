# DECIFRA - ESCOPO MVP COMPLETO
## Documento para Replit Agent

---

## 1. VISÃO EXECUTIVA

**O que é:** Aplicativo mobile de avaliação de personalidade Big Five (IPIP-NEO-120) para treinadoras comportamentais e suas clientes.

**Propósito do MVP:** Permitir que treinadoras comprem créditos, gerem códigos, clientes respondam o teste de 120 questões, e ambos visualizem resultados com protocolos recomendados.

**Diferencial:** Sistema inteligente de 90 protocolos que cruza as 30 facetas para recomendar planos de ação personalizados.

**Públicos:** 
- Treinadoras (login): gerenciam tudo no app
- Clientes (código): fazem teste e veem resultado básico
- Admin: visão geral (web, fora do MVP)

---

## 2. ARQUITETURA TÉCNICA MVP

### Stack Definida
```
FRONTEND: React Native (Expo) - app mobile iOS/Android
BACKEND: Supabase (PostgreSQL + Auth + Edge Functions)
EMAIL: Resend (envio de códigos)
BUILD: EAS Build (Expo Application Services)
```

### Por que essa stack
- React Native = uma codebase, dois apps
- Supabase = backend completo sem servidor próprio
- Edge Functions = processa webhook Hotmart e cálculos
- Expo = build fácil, preview no celular

## 3. ESTRUTURA DE DADOS (TABELAS SUPABASE)

### 3.1 treinadoras
```sql
id: uuid (primary key, from auth.users)
nome: text (required)
email: text (unique, required)
telefone: text
creditos_disponiveis: integer (default 0)
status: text (ativo/inativo)
created_at: timestamp
```

### 3.2 codigos
```sql
codigo: text (primary key, format: ART-XXXX)
treinadora_id: uuid (foreign key → treinadoras)
status: text (ativo/usado/expirado)
usado_por: uuid (foreign key → clientes, nullable)
created_at: timestamp
expires_at: timestamp (default: now + 30 days)
```

### 3.3 clientes
```sql
id: uuid (primary key)
nome: text (required)
email: text (nullable)
codigo_usado: text (foreign key → codigos)
treinadora_id: uuid (foreign key → treinadoras)
created_at: timestamp
teste_completado: boolean (default false)
```

### 3.4 respostas
```sql
id: uuid (primary key)
cliente_id: uuid (foreign key → clientes)
item_numero: integer (1-120)
resposta: integer (1-5)
fator: text (N/E/O/A/C)
faceta: integer (1-6)
invertido: boolean
created_at: timestamp
```

### 3.5 resultados
```sql
id: uuid (primary key)
cliente_id: uuid (foreign key → clientes)
tipo: text (fator/faceta)
codigo: text (N, N1, E, E3, etc.)
nome: text
score_bruto: integer
percentil: integer (5-95)
classificacao: text (Muito Baixo/Baixo/Medio/Alto/Muito Alto)
created_at: timestamp
```

### 3.6 protocolos
```sql
id: text (primary key, ex: N1-A, N1-B, N1-C)
nome: text
descricao: text
objetivo: text
exercicios: jsonb
fator: text (N/E/O/A/C)
faceta: integer (1-6)
tipo: text (polo_alto/polo_baixo/ajuste_fino)
```

### 3.7 protocolos_recomendados
```sql
id: uuid (primary key)
cliente_id: uuid (foreign key → clientes)
protocolo_id: text (foreign key → protocolos)
prioridade: integer (1-6)
tipo_exibicao: text (cliente/treinadora)
created_at: timestamp
```

## 4. FLUXOS DO MVP

### FLUXO A: TREINADORA (com login)

```
A1. LOGIN
    └── Email + senha → Supabase Auth
    └── Sucesso → Dashboard

A2. DASHBOARD
    └── Mostra: créditos disponíveis, clientes recentes, ações rápidas
    └── Botões: [Gerar código] [Ver clientes] [Comprar créditos]

A3. GERAR CÓDIGO
    └── Form: nome da cliente (obrigatório), email (opcional)
    └── Valida: tem créditos disponíveis?
    └── Gera: código único ART-XXXX
    └── Mostra: código + botão copiar + botão compartilhar
    └── Decrementa: créditos da treinadora

A4. LISTA DE CLIENTES
    └── Mostra: todas as clientes da treinadora
    └── Status: código ativo / teste em andamento / teste completo
    └── Filtros: todas / ativas / completadas

A5. RESULTADO COMPLETO (treinadora)
    └── Acesso: quando cliente completou teste
    └── Mostra: 
        ├── Mandala dos 5 fatores (visual)
        ├── Scores numéricos (percentis)
        ├── 6 protocolos recomendados (3 principais + 2 compl. + 1 ajuste)
        └── Botão: exportar PDF

A6. PERFIL
    └── Dados da treinadora
    └── Botão: logout
```

### FLUXO B: CLIENTE (com código)

```
B1. ENTRADA POR CÓDIGO
    └── Tela: "Descubra sua arquitetura emocional"
    └── Input: código ART-XXXX
    └── Valida: código existe? está ativo? não expirado?
    └── Sucesso → Cadastro rápido

B2. CADASTRO RÁPIDO
    └── Campos: nome (obrigatório), email (opcional), idade (opcional)
    └── Sucesso → Introdução ao teste

B3. TESTE IPIP-NEO-120 (4 ESTAÇÕES)
    └── Estação 1: "Como você pensa" (itens 1-30)
    └── Estação 2: "Como você age" (itens 31-60)
    └── Estação 3: "Como você conecta" (itens 61-90)
    └── Estação 4: "Como você reage" (itens 91-120)
    └── Cada item: escala 1-5 (Discordo totalmente → Concordo totalmente)
    └── Persistência: salva a cada resposta (pode sair e voltar)

B4. PROCESSANDO
    └── Tela: "Decifrando seu perfil..."
    └── Animação: mandala se formando
    └── Background: calcula scores, gera resultados, recomenda protocolos

B5. RESULTADO CLIENTE
    └── Mostra:
        ├── Mandala dos 5 fatores (sem números, só visual)
        ├── Texto: "Quem você é" (interpretação do perfil)
        ├── 4 protocolos (3 principais + 1 ajuste fino) - só nomes
        └── Call-to-action: "Agende com sua treinadora"
    └── NÃO mostra: percentis, scores, protocolos completos
```

### FLUXO C: INTEGRAÇÃO HOTMART (automático)

```
C1. WEBHOOK HOTMART
    └── Endpoint: /webhook/hotmart
    └── Recebe: dados da compra (product_id, buyer_email, etc.)
    └── Verifica: email comprador = email treinadora cadastrada?
    └── Se sim: adiciona créditos à treinadora
    └── Envia: email com confirmação da compra

C2. EDGE FUNCTION SUPABASE
    └── Processa: webhook
    └── Valida: assinatura do Hotmart
    └── Atualiza: tabela treinadoras (creditos_disponiveis)
    └── Insere: registro na tabela compras
```

## 5. ALGORITMOS ESSENCIAIS

### 5.1 Cálculo de Scores (após 120 respostas)

```typescript
// Passo 1: Inverter itens com direction "-"
// resposta_invertida = 6 - resposta_original

// Passo 2: Calcular score bruto por faceta (4 itens)
// score_faceta = soma das 4 respostas (já invertidas)
// range: 4-20

// Passo 3: Calcular score bruto por fator (6 facetas = 24 itens)
// score_fator = soma das 6 facetas
// range: 24-120

// Passo 4: Converter para percentil 5-95%
// percentil_faceta = 5 + ((score - 4) / 16) * 90
// percentil_fator = 5 + ((score - 24) / 96) * 90

// Passo 5: Classificar em 5 faixas
// Muito Baixo: 5-22%
// Baixo: 23-39%
// Medio: 40-56%
// Alto: 57-73%
// Muito Alto: 74-95%
```

### 5.2 Recomendação de Protocolos

```typescript
// Passo 1: Pontuar cada faceta
// Muito Alto/Baixo = 3 pontos
// Alto/Baixo = 2 pontos
// Medio = 0 pontos

// Passo 2: Bônus de segurança
// Se N1, N5, ou N6 estiverem altos: +1 ponto

// Passo 3: Verificar combinações críticas
// Ex: N alto + C5 baixo → prioriza N6-A e C5-B

// Passo 4: Ordenar por pontuação (maior primeiro)

// Passo 5: Selecionar
// Cliente: top 3 + 1 ajuste fino (4 total)
// Treinadora: top 3 + 2 complementares + 1 ajuste (6 total)

// Passo 6: Ordenar por prioridade temática
// 1. Regulação/Estabilização (N)
// 2. Execução/Consistência (C)
// 3. Limites/Relação (A/E)
// 4. Energia social (E)
// 5. Visão/Criatividade (O)
```

## 7. DADOS INICIAIS (seed)

### 7.1 120 Itens do Teste
- Lista completa fornecida em documento
- Cada item: id, texto, fator, faceta, direction (+ ou -)
- Ordem de apresentação: fixa (array de 120 posições)

### 7.2 90 Protocolos
- Estrutura: 3 por faceta (polo alto, polo baixo, ajuste fino)
- IDs: N1-A, N1-B, N1-C, N2-A, ... C6-C
- Conteúdo: nome, descrição, objetivo, exercícios (JSON)

### 7.3 Textos Interpretativos
- 25 textos de fatores (5 fatores × 5 faixas)
- 60 textos de polos (30 facetas × 2 polos)
- 30 descrições curtas das facetas

## 8. REGRAS DE NEGÓCIO

### 8.1 Códigos
- Formato: ART-XXXX (4 caracteres alfanuméricos maiúsculos)
- Validade: 30 dias após geração
- Uso: único, após uso vira "usado"
- Custo: 1 crédito por código

### 8.2 Créditos
- Compra: fora do app (Hotmart), via web
- Entrega: webhook automático adiciona à conta
- Uso: geração de códigos consome 1 crédito cada

### 8.3 Teste
- Obrigatório: 120 respostas
- Persistência: salva localmente (pode retomar)
- Tempo mínimo: 8 minutos (alerta se menor)
- Padrão suspeito: alerta se 100% iguais

### 8.4 Resultados
- Cliente: visual básico, sem números, 4 protocolos
- Treinadora: completo, com números, 6 protocolos, exporta PDF
